На сайте есть всегда изображения. Они помогают всегда донести смысл или просто добавить декоративное оформление страницы. Есть 2 типа изображения страницы - контентное и декоративное. Какая между ними разница?

Контентное изображение - несут смысловую нагрузку, относится к содержанию страницы и помогают донести до пользователя полезную ему инфу. Они добавляются в HTML - используя тег img, с обязательным заполненным атрибутом alt, описывающим изображение.

Фотография продукта на карточке товара в интернет магазине, фотографии членов команды, лого компании, рекламные баннеры, карты с локацией, картинки слайдеров, аватарки - это все контентное изображение


Декоративное изображение - используется ислючительно для доп оформления, они не несут никакой инфы и служат только для красоты. Добавляется через css как фон. Использует свойство background-image, или размечается в HTML тегом <svg> (в случае векторной графики). Кастомные маркеры списков, всевозможные вспомогательные иконки, плашки, фоновые изображения блоков - это все декоративные изображения.
Несколько особенностей такого разделения
Очищается код страницы, что улучшает его читабельность для разработчика, упрощает работу поисковых роботов и ускоряет начальную загрузку страницы.
Все, что должно быть проиндексировано роботами добавлям как <img>, остальное фон.
При печати веб страницы теги <img> печатаются, а фоновые изображения нет.
Ассистивные технологии (скринридеры) для людей с ограничеными возможностями читают <alt> изображений, поэтому если изображение несет инфу, оно обязательно должно быть <html> разметки.

Свойство background-color

Задает цвет фона элемента. Значение может быть зарезервированое слово или цвет, записаный в <rgb>, <rgba> и <hex> формате
background-color: цвет;

Свойство background-image

Задает фоновое изображение элемента, адрес которого указывается аргументами <url()>. Это может быть как относительный так и абсолютный путь.

background-image: url("адрес картинки")

Можно сразу задать цвет фона и фоновое изображение, которые будут поверх фона. Например, в случае когда не загрузилась фоновая картинка пользователь увидит хотя бы цвет фона а не пустое место.
body {
background-color: #fafafa;
}
.hero {
background-image: url(https://photo/1280.jpg);
}

Свойство background-repeat

Управляет повторением фонового изображения. По умолчанию оно повторяется по горизонтали и вертикали. Это заметно, если картинка меньше чем размер элемента.
background-repeat: repeat | repeat-x | repeat-y | no-repeat

  • repeat - повторять по X и Y. Значение по умолчанию.
  • repeat-x - повторять только по X, то есть по горизонтали.
  • repeat-y - повторять только по Y, то есть по вертикали.
  • no-repeat - не повторять.

Свойство background-position

Управляет положением фонового изображения относительно границ элемента, используя для координат - х (по горизонтали) и у (по вертикали).
background-position: x y;
В качестве значения можно использовать абсолютные еденицы или относительные еденицы и ключевые слова (top, bottom,left, rihgt, center).
По умолчанию задано значение left для х и top для у, т.е фон позиционируется относительно верхнего левого угла элемента.

Поставить фоновое изображение по центру
background-position: 50% 50%
Поставить фоновое изображение 100px от левого края и 200px от верха background-position: 100px 200px
Поставить изображение в нижний правый угол background-position: right bottom
Поставить изображение в верхний левый угол background-position: left top

Позиция со смещением

Чтобы не привязываться к размерам элементов и не задавать позицию фона жестко, можно использовать синтаксис позиций со смещением
background-position: x-смещение у-смещение

Например, если необходимо поставить фон в нижний правый угол элемента в произвольных размераз так, что бы фон отставал от границ на 30px справа, 20px снизу
background-position: right 30px bottom 20px

background-size

По умолчани. браузер не изменяет размер фонового изображения. Если картинка будет больше элемента, будет пустое пространство, не занятое фоном, фон будет виден только частично.
background-size: auto значение cover contain
Это свойство позволяет подкорректировать размеры изобрадения, указав необходимую ширину и высоту. Если не указать высоту, она будет вычисляться автоматически, сохраняя пропорции.

    Мсходные размеры изображения, значения по умолчанию

  • background-size: auto auto

    • Ширина 200px, высота произвольная

  • background-size: 200px

    • Ширина 200px, высота 300px

  • background-size: 200px 300px

Значение contain и cover

Знгачения так же может быть 2 зарезервированых слова contain и color. Они стремятся вместить в контейнер фоновое изображение по максимуму, только одна за счет образания лишнего, а другое за счет появления пустого пространства в контейнере
Значение contain

  • Гарантировано сохраняет пропорции изображения
  • Изображению задаются максимально возможные размеры (не больше оригинала), при которых оно полностью помещается в элемент.
  • Изображение может не закрыть весь фон элемента по вертикали или горизонтали, если пропорции блока и изображения не совпадают.

Значение cover

  • Гарантировано сохраняет пропорции изображения
  • Изобранию задается минимальные размеры, при которых оно зальет фон всего элемента
  • Если пропорции изображения и элемента разные, часть изображения, по вертикали или горизонтали визуально обрезается

Свойство background-attachment

По умолчанию фоновое изображение прокручивается вместе с содержимым элемента. С помощью свойства background-attachment можно зафиксировать фон на месте и запретить прокручиваться при скролле

background-attachment: scroll | fixed | inherit

  • scroll - фон прокручивается вместе с содержимым. Это значение по умолчанию.
  • fixed - фон не прокручивается, зафиксирован на месте.

Многослойный фон

Элементу можно задать несколько фоновых изображений одновременно, достаточно перечислисть их свойство background-image через запятую. Для каждого изображения так же можно задать значение других фонов, тоже через запятую в каждом свойстве.

  • background-image: url("картинка1"), url("картинка2")
  • background-size: 100px cover
  • background-position: top right, center
  • background-repeat: repeat-x, no-repeat

В большинстве случаев многослойный фон можно заменить одним изображением, на котором в специальном редакторе были совмещены все другие элементы. Вес такого файла будет меньше, чем общий вес отдельных картинок, и браузеру будет необходимо сделать 1 запрос вместо 4, что так же увеличит скорость загрузки отображения страницы.
